CodeBrix.Python is a cross-platform Python and .NET language-interoperability
library. It is a faithful port of the Python.NET (pythonnet) project, version
3.1.0, re-namespaced to CodeBrix.Python.* and retargeted to .NET 10.

CodeBrix.Python incorporates source code adapted from the open-source project
listed below, distributed under the MIT License, which is compatible with the
MIT License of CodeBrix.Python itself. The original copyright notice and
permission notice are reproduced in full below, as the MIT License requires
them to travel with the source.

For every incorporated file, the C# `namespace` line carries a provenance
comment of the form `//was previously: <original.namespace>;` recording the
upstream namespace the file was adapted from.

================================================================================
1. Python.NET (pythonnet)
================================================================================

Project:    Python.NET (pythonnet)
Source URL: https://github.com/pythonnet/pythonnet
Version:    3.1.0 (git tag v3.1.0, commit 3d76836e80284aae8abcac92e1676d7d83178ec1)
Authors:    The Contributors of the Python.NET Project
License:    MIT

Incorporated source (production):
    The entire src/runtime project (the Python.Runtime assembly) - the managed
    Python<->CLR interop engine - was adapted into the CodeBrix.Python library,
    including the embedded Python resources clr.py, interop.py, and the
    Mixins/*.py files (carried verbatim).

Incorporated source (tests):
    The test projects were adapted into tests/:
      - src/embed_tests (Python.EmbeddingTest, NUnit) -> CodeBrix.Python.Tests,
        converted to xUnit v3 + SilverAssertions.
      - src/testing (Python.Test, the C# support assembly that Python imports
        during tests) -> CodeBrix.Python.TestSupport.
      - src/python_tests_runner + the tests/*.py pytest suite ->
        CodeBrix.Python.PythonTests, which carries the Python test files (under
        pytests/, with their .NET namespace references renamed), conftest, and
        fixtures, plus an in-process pytest runner.
    Test code is not redistributed in the NuGet package.

Modifications made during the port:
    - Namespaces renamed from Python.Runtime.* to CodeBrix.Python.* (see the
      provenance comment on each file's namespace line); the assembly was
      renamed from Python.Runtime to CodeBrix.Python. The Python-facing module
      name (clr) is unchanged, preserving the public Python API.
    - Retargeted from netstandard2.0 to net10.0; the netstandard-era polyfill
      package references (System.Reflection.Emit, Lost.Compat.NullabilityAttributes,
      Microsoft.CSharp) were dropped because those APIs are built into .NET 10.
    - Strong-name signing (pythonnet.snk) was removed to match the CodeBrix family
      convention; InternalsVisibleTo entries were updated to the renamed,
      unsigned test assemblies.
    - The original Properties/AssemblyInfo.cs assembly attributes are now
      expressed via MSBuild csproj properties.
    - The AppDomain-reload test project (tests/domain_tests) was NOT incorporated:
      AppDomain unload/reload is a .NET-Framework concept that does not apply to a
      .NET 10-only library.
    - block-scoped namespaces converted to file-scoped; nullable-reference-type
      annotations were PRESERVED (the interop code depends on them).
    - The upstream's formatter-based serialization API (RuntimeData / IFormatter /
      BinaryFormatter / ISerializable) and Code-Access-Security attributes were
      kept for API parity; on .NET 10 these are obsolete/inert (the engine falls
      back to a NoopFormatter at run time). FormatterServices.GetUninitializedObject
      was replaced with the supported RuntimeHelpers.GetUninitializedObject.
